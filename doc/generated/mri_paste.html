Overview


  mri_paste takes a number of pgh MRI datasets of any type 
  and outputs a dataset of the same type containing the data
  from all the files concatenated on one of the dimensions of
  the data.  

  The command line for mri_paste is:
    mri_paste [-dimension v|x|y|z|t] [-outfile ofile] [-verbose] 
       infile1 infile2 [infile3 ... [infileN]]

  or:
    mri_paste -help

Examples


  mri_paste infile1 infile2

  This use takes all the defaults:
         -d t (time dimension)
         -out mri_paste_out

  mri_paste -d z -out slice_extended -v slice1 slice2 slice3

  This use takes the files containing slices 1, 2, and 3 and
  concatenates them into one file, slice_extended, on dimension z. The
  verbose flag is turned on to display the progress of mri_paste.

Arguments

dimension

  [-dimension v|x|y|z|t]	(-d v|x|y|z|t)

  Ex. -d t
 
  The value of dimension is a single character that specifies the
  dimension along which to concatenate the input files.  Valid values
  are one character from the dimension string of the input data
  (typically vxyzt); case is ignored. Default is "t".

outfile

  [-outfile Outfile]	(-out Outfile)

  Ex. -out slice_extended

  Outfile specifies the output dataset. Default is "mri_paste_out".

verbose

  [-verbose]	(-v)

  Ex. -v

  Sets the verbose flag to on, which displays output to the screen
  showing progress of mri_paste.

Details

Inputs


  There must be at least two input files; however, the same input file
  can be used more than once.

  Input files must all have the same chunks.

  Input files must all have the same dimension order.

  Input files must all have the selected dimension (this can be added
  with mri_remap if necessary).

  The lengths of the non-selected dimensions must be the same in all
  input files.

  The length of the selected dimension in the output file will be the
  sum of that dimension's extents in all the input files.

  The outfile is not allowed to have the same name as any of the input
  files.
